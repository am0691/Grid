# GRID 웹서비스 PRD - 사용자 경험

## 1. 사용자 여정 (User Journey)

### 1.1 초기 진입 및 설정
양육자는 처음 시스템에 접속하여 양육 대상자(영혼)를 등록하고 양육 프로그램을 시작합니다.

**주요 단계:**
1. **로그인 및 대시보드 접근**
   - 양육자가 웹 서비스에 로그인하여 개인 대시보드를 확인합니다.

2. **새 영혼 등록**
   - 양육 대상자의 기본 정보(이름, 연락처 등)를 입력합니다.
   - 양육 프로그램 유형을 선택합니다 (CONVERT: 새신자 13주, DISCIPLE: 제자훈련 12개월 등).

3. **양육 계획 생성**
   - 시스템이 선택한 프로그램에 맞는 GRID 템플릿을 자동 생성합니다.
   - 시작일을 설정하면 각 주차/월별 일정이 자동으로 계산됩니다.

### 1.2 일상적 양육 관리
양육자는 정기적으로 시스템에 접속하여 양육 진행 상황을 기록하고 확인합니다.

**주요 단계:**
1. **대시보드에서 영혼 선택**
   - 양육 중인 영혼 목록에서 해당 영혼을 클릭하여 그리드 뷰로 이동합니다.

2. **그리드 뷰에서 진도 확인**
   - 각 영역(구원의 확신, 말씀, 교제, 기도 등)별 진행 상황을 시각적으로 파악합니다.
   - 완료된 항목(연한 색상), 현재 진행 중인 항목(진한 색상+테두리), 미래 항목(회색)을 한눈에 확인합니다.

3. **완료 체크 및 메모 입력**
   - 양육 후 해당 주차/월의 셀을 클릭하여 완료 표시를 합니다.
   - 간단한 메모나 특이사항을 기록합니다 (예: "성경 암송 완료", "개인 간증 나눔", "다음 주 복습 필요").

4. **전체 진도 모니터링**
   - 각 영역별 완료율을 확인하여 균형 잡힌 양육이 이루어지는지 점검합니다.
   - 지연되고 있는 영역을 식별하여 다음 만남에 집중할 항목을 계획합니다.

### 1.3 지속적 양육 및 완료
양육자는 프로그램이 끝날 때까지 꾸준히 진도를 관리하고, 완료 후에는 기록을 보관합니다.

**주요 단계:**
1. **정기적 리뷰**
   - 주차별/월별로 진행 상황을 확인하고 다음 단계를 준비합니다.
   - 필요시 이전 주차의 메모를 다시 확인하여 연속성 있는 양육을 제공합니다.

2. **프로그램 완료**
   - 모든 영역의 모든 주차/월이 완료되면 시스템이 자동으로 완료 상태로 표시합니다.
   - 전체 양육 기간 동안의 요약 보고서를 생성할 수 있습니다.

3. **다음 단계 진행**
   - CONVERT 프로그램 완료 후 DISCIPLE 프로그램으로 전환할 수 있습니다.
   - 완료된 기록은 아카이브되어 언제든지 조회 가능합니다.

## 2. 핵심 화면 설명

### 2.1 대시보드 (Dashboard)
**목적:** 양육자가 관리하는 모든 영혼의 현황을 한눈에 파악

**주요 요소:**
- **영혼 목록 카드**
  - 각 영혼별로 카드 형태로 표시
  - 카드 내용: 이름, 프로그램 유형, 시작일, 전체 진도율(%)
  - 현재 진행 중인 주차/월 표시

- **빠른 필터 및 정렬**
  - 프로그램 유형별 필터 (CONVERT, DISCIPLE 등)
  - 진도율 순, 시작일 순, 이름 순 정렬

- **새 영혼 추가 버튼**
  - 화면 우측 상단에 고정 배치
  - 클릭 시 영혼 등록 양식으로 이동

**사용자 행동:**
- 대시보드는 로그인 직후 첫 화면으로 표시됩니다.
- 양육자는 카드를 클릭하여 해당 영혼의 상세 그리드 뷰로 이동합니다.

### 2.2 그리드 뷰 (Grid View)
**목적:** 특정 영혼의 양육 진도를 영역별, 기간별로 시각화하여 관리

**주요 요소:**
- **헤더 영역**
  - 영혼 이름 및 기본 정보
  - 프로그램 유형 및 기간 표시
  - 전체 진도율 게이지 또는 퍼센트

- **그리드 테이블**
  - **행(세로축):** 주차(1~13주) 또는 월(1~12월)
  - **열(가로축):** 양육 영역 (예: 구원의 확신, 말씀, 기도, 교제, 증거, 주재권, 세계비전, 양육, 성품 등)
  - **셀 상태 시각화:**
    - **완료 상태:** 연한 색상 (예: 연한 녹색, 파란색, 노란색 - 영역별 고유 색상 사용)
    - **현재 진도:** 진한 색상 + 두꺼운 테두리 (예: 별(★) 아이콘 표시)
    - **미완료 상태:** 회색 또는 흰색 배경
  - **셀 내용:** 각 셀에는 해당 주차/월의 학습 주제나 활동 키워드가 텍스트로 표시됨

- **영역별 색상 코드**
  - 각 영역마다 고유한 색상을 부여하여 시각적 구분을 명확히 함
  - 예: 구원의 확신(녹색), 말씀(파랑), 교제(노랑), 기도(주황) 등

- **완료 체크 인터랙션**
  - 셀 클릭 시 모달 창 또는 사이드 패널 열림
  - 완료 체크박스, 날짜 선택기, 메모 입력란 제공
  - 저장 버튼 클릭 시 셀 상태가 "완료"로 변경되고 색상이 업데이트됨

**사용자 행동:**
- 양육자는 그리드를 스캔하여 어느 영역이 잘 진행되고 있는지, 어느 영역이 지연되고 있는지 빠르게 파악합니다.
- 현재 진행 중인 셀(★ 표시)을 중심으로 다음 양육 내용을 확인합니다.
- 양육 후 해당 셀을 클릭하여 완료 체크와 메모를 남깁니다.

### 2.3 영혼 상세 정보 (Soul Detail)
**목적:** 특정 영혼의 기본 정보 및 양육 히스토리 관리

**주요 요소:**
- **기본 정보 편집**
  - 이름, 연락처, 양육 시작일 등 수정 가능

- **메모 히스토리**
  - 각 주차/월별로 입력한 메모들을 시간 순으로 나열
  - 특정 메모 클릭 시 해당 그리드 셀로 이동

- **전체 진도 요약**
  - 영역별 완료율을 막대 그래프나 도넛 차트로 시각화
  - 양육 기간 타임라인 표시

**사용자 행동:**
- 그리드 뷰 상단의 "상세 정보" 버튼을 통해 접근합니다.
- 양육 대상자의 연락처나 특이사항을 확인하거나 수정합니다.
- 과거 메모를 검토하여 양육의 연속성을 유지합니다.

### 2.4 영혼 등록 (New Soul Registration)
**목적:** 새로운 양육 대상자를 시스템에 추가

**주요 요소:**
- **입력 폼**
  - 필수 항목: 이름, 프로그램 유형 선택
  - 선택 항목: 연락처, 시작일(기본값: 오늘), 초기 메모

- **프로그램 선택**
  - 드롭다운 메뉴 또는 라디오 버튼으로 프로그램 유형 선택
  - 각 프로그램에 대한 간단한 설명 툴팁 제공

- **확인 및 저장**
  - "등록" 버튼 클릭 시 대시보드로 돌아가며 새 영혼 카드가 추가됨

**사용자 행동:**
- 대시보드의 "새 영혼 추가" 버튼을 클릭하여 접근합니다.
- 필수 정보를 입력하고 프로그램을 선택한 후 등록합니다.
- 등록 즉시 해당 영혼의 그리드 뷰로 이동하거나 대시보드로 돌아갑니다.

## 3. 접근성 및 사용성 원칙

### 3.1 접근성 (Accessibility)
**기본 방침:** 다양한 연령대와 기술 수준의 사용자가 불편 없이 사용할 수 있도록 설계

**구체적 원칙:**
- **명확한 시각적 구분**
  - 높은 명암 대비를 사용하여 텍스트와 배경이 명확히 구분되도록 함
  - 색상뿐만 아니라 아이콘(✓, ★ 등)과 텍스트 레이블을 병행하여 색맹 사용자도 상태를 구분할 수 있도록 함

- **큰 클릭 영역**
  - 셀 및 버튼의 클릭 영역을 충분히 크게 설정하여 터치 인터페이스(모바일)에서도 정확한 선택이 가능하도록 함
  - 최소 44x44픽셀의 터치 타겟 권장

- **명확한 피드백**
  - 버튼 클릭, 셀 완료 체크 등 모든 인터랙션에 즉각적인 시각적 피드백 제공
  - 로딩 중일 때는 스피너나 진행 표시를 명확히 표시

- **반응형 디자인**
  - PC 화면을 기본으로 하되, 태블릿 및 모바일 환경에서도 기본 기능 사용 가능
  - 모바일에서는 그리드 테이블을 좌우 스크롤 가능하도록 설계

### 3.2 사용성 (Usability)
**기본 방침:** 직관적이고 학습 곡선이 낮은 인터페이스 제공

**구체적 원칙:**
- **일관된 네비게이션**
  - 모든 화면에서 "대시보드로 돌아가기" 버튼을 동일한 위치에 배치
  - 브레드크럼(Breadcrumb) 네비게이션을 제공하여 현재 위치를 명확히 표시

- **최소한의 클릭**
  - 주요 기능(영혼 선택 → 그리드 뷰 → 완료 체크)에 도달하기까지 3클릭 이내로 제한
  - 빠른 작업을 위해 대시보드에서 바로 최근 영혼에 접근 가능한 숏컷 제공

- **명확한 시각적 계층**
  - 중요한 정보(현재 진도, 완료율)를 화면 상단이나 눈에 띄는 위치에 배치
  - 보조 정보(메모, 상세 정보)는 필요시 접근할 수 있도록 하위 계층에 배치

- **상태 표시의 직관성**
  - 색상, 아이콘, 텍스트를 조합하여 셀의 상태(완료/진행 중/미완료)를 한눈에 알 수 있도록 함
  - 범례(Legend)를 화면 상단이나 사이드바에 고정 표시하여 색상 의미를 언제든 확인 가능

- **오류 방지 및 복구**
  - 삭제나 수정 같은 중요한 작업 전에 확인 다이얼로그 표시
  - 실수로 완료 체크를 취소해야 할 경우 쉽게 되돌릴 수 있는 기능 제공

- **도움말 및 온보딩**
  - 첫 사용자를 위한 간단한 튜토리얼 제공 (선택적 스킵 가능)
  - 주요 기능에 툴팁을 제공하여 마우스 오버 시 간단한 설명 표시

### 3.3 성능 및 반응성
**기본 방침:** 빠른 로딩과 즉각적인 반응으로 사용자 경험 향상

**구체적 원칙:**
- **빠른 초기 로딩**
  - 대시보드는 2초 이내에 로딩 완료
  - 그리드 뷰는 3초 이내에 렌더링 완료

- **즉각적인 인터랙션**
  - 셀 클릭, 완료 체크 등 모든 인터랙션은 1초 이내에 피드백 제공
  - 네트워크 지연 시에도 낙관적 업데이트(Optimistic Update)를 통해 즉각적인 UI 반응 제공

### 3.4 데이터 보안 및 신뢰성
**기본 방침:** 양육 데이터의 안전한 보관 및 복구 가능성 보장

**구체적 원칙:**
- **자동 저장**
  - 메모 입력 및 완료 체크는 자동으로 저장되어 데이터 손실 방지

- **백업 및 복구**
  - 모든 데이터는 서버에 안전하게 저장되며 사용자는 언제든지 이전 상태로 복구 가능

- **개인정보 보호**
  - 양육 대상자의 개인정보는 암호화되어 저장
  - 양육자만 자신이 관리하는 영혼의 데이터에 접근 가능

## 4. 사용자 여정 시나리오 예시

### 시나리오: 양육자 김목사가 새신자 이영희를 13주 CONVERT 프로그램으로 양육하는 과정

**1주차 일요일 (프로그램 시작)**
1. 김목사가 로그인하여 대시보드 접근
2. "새 영혼 추가" 클릭
3. 이름: "이영희", 프로그램: "CONVERT (13주)", 시작일: "2026-01-29" 입력 후 등록
4. 이영희의 그리드 뷰가 생성되어 1주차 "구원의 확신" 영역이 현재 진도(★)로 표시됨

**1주차 수요일 (첫 양육 모임 후)**
1. 대시보드에서 "이영희" 카드 클릭
2. 그리드 뷰에서 1주차 "구원의 확신" 셀 클릭
3. 완료 체크박스 선택, 메모: "다리 예화 이해 잘함, 구원 간증 작성 돕기 완료" 입력 후 저장
4. 셀이 연한 녹색으로 변경되고 체크 표시(✓) 아이콘 추가됨
5. 자동으로 다음 셀(1주차 "말씀")이 현재 진도(★)로 표시됨

**3주차 목요일 (중간 점검)**
1. 대시보드에서 이영희의 진도율 확인: "구원의 확신 100%, 말씀 67%, 교제 33%"
2. 그리드 뷰에서 "교제" 영역이 지연되고 있음을 시각적으로 파악
3. 다음 모임에서 "교제" 영역에 집중하기로 계획

**13주차 일요일 (프로그램 완료)**
1. 마지막 셀 완료 체크 후 시스템이 자동으로 "프로그램 완료" 메시지 표시
2. 전체 양육 요약 보고서를 다운로드하거나 출력
3. 대시보드에서 이영희 카드에 "완료" 배지 표시
4. 선택적으로 DISCIPLE 프로그램으로 전환 가능

## 5. 핵심 사용자 경험 가치

**1. 시각적 명확성**
- 한눈에 전체 양육 진도를 파악할 수 있어 체계적인 관리 가능
- 색상과 아이콘을 활용한 직관적인 상태 표시

**2. 효율적인 기록 관리**
- 간단한 클릭과 메모 입력만으로 양육 내용을 체계적으로 기록
- 과거 기록을 언제든지 조회하여 연속성 있는 양육 제공

**3. 동기 부여**
- 진도율 게이지와 완료 체크를 통해 성취감 제공
- 영역별 균형 있는 양육을 시각적으로 확인하며 동기 부여

**4. 접근성과 유연성**
- 다양한 연령대와 기술 수준의 양육자가 쉽게 사용 가능
- PC, 태블릿, 모바일 등 다양한 환경에서 접근 가능

**5. 신뢰성**
- 자동 저장과 백업으로 데이터 손실 걱정 없음
- 오랜 기간 축적된 양육 데이터를 안전하게 보관
