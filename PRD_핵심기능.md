# GRID 웹서비스 PRD - 핵심 기능

## 1. 핵심 기능 개요

GRID는 양육자가 영혼(Convert 또는 Disciple)의 영적 성장을 체계적으로 관리하고 추적할 수 있는 영역별 진도 관리 시스템입니다.

---

## 2. 주요 기능 목록

### 2.1 영혼 관리

**기능 설명**
- 양육자가 관리하는 영혼(Convert 또는 Disciple) 목록을 생성하고 관리
- 각 영혼에 대한 기본 정보 입력 (이름, 시작일, 양육 유형)
- 영혼별로 독립적인 GRID 진도 현황 관리

**사용자 가치**
- 여러 명의 영혼을 동시에 양육할 때 각자의 진도를 명확히 구분하여 관리
- 각 영혼의 성장 과정을 개별적으로 추적하고 맞춤형 양육 계획 수립 가능
- 새로운 영혼 추가 시 자동으로 해당 양육 유형에 맞는 GRID 생성

**사용 시나리오**
```
김철수 양육자는 3명의 Convert와 2명의 Disciple을 양육하고 있습니다.
- Convert: 이영희(1주차), 박민수(3주차), 최지훈(2주차)
- Disciple: 정수진(5개월차), 한민준(8개월차)

각 영혼의 현재 진도를 한눈에 확인하고, 이번 주에 누구와 어떤 영역을
다룰지 계획할 수 있습니다.
```

---

### 2.2 영역별 독립 진도 관리

**기능 설명**
- GRID의 각 영역(구원의 확신, 말씀, 교제, 증거 등)마다 독립적인 진도 설정
- 한 영혼이 각 영역에서 서로 다른 주차/월차 진행 가능
- 영역별로 현재 주차/월차 표시 및 완료 여부 체크

**사용자 가치**
- **유연한 양육 진행**: 모든 영역을 동일한 속도로 진행할 필요 없음
- **개인 맞춤형 양육**: 각 영혼의 강점과 약점에 따라 특정 영역에 집중 가능
- **현실적인 진도 관리**: 특정 영역이 지연되어도 다른 영역은 계속 진행

**사용 시나리오**
```
이영희의 양육 현황:
- 구원의 확신: 3주차 완료 (잘 이해하고 있음)
- 말씀: 1주차 완료 (아직 초기 단계)
- 교제: 3주차 완료 (적극적으로 참여)
- 죄에서 떠남: 2주차 완료 (좀 더 시간이 필요함)

김 양육자는 이영희가 말씀 영역을 천천히 진행하더라도, 교제와 구원의
확신 영역은 계획대로 진행할 수 있습니다. 이를 통해 각 영역의 실제
이해도에 맞춰 속도를 조절할 수 있습니다.
```

---

### 2.3 그리드 뷰 (Grid View) - 전체 진도 시각화

**기능 설명**
- 세로축: 주차/월차 (Convert 13주, Disciple 12개월)
- 가로축: 영역 (Convert 5개, Disciple 14개)
- 각 셀에 해당 주차/월차의 양육 내용 표시
- 완료된 셀은 색상으로 표시하여 진도 시각화

**사용자 가치**
- **한눈에 전체 파악**: 12개월 또는 13주 전체 양육 과정을 한 화면에서 조망
- **직관적인 진도 확인**: 어떤 영역이 빠르게/느리게 진행되는지 색상으로 즉시 인식
- **계획 수립 용이**: 앞으로 다룰 내용을 미리 보고 준비 가능

**사용 시나리오**
```
김 양육자가 박민수(Convert 3주차)의 GRID를 엽니다.

화면에 13주 x 5개 영역의 그리드가 표시됩니다:
- 구원의 확신: 1~3주 완료(초록색), 4~13주 예정(회색)
- 말씀: 1~2주 완료(파란색), 3주 진행중(연파란색), 4~13주 예정
- 교제: 1주만 완료(노란색), 2~13주 예정
- 죄에서 떠남: 1~2주 완료(보라색), 3~13주 예정
- 참고사항: 아직 시작 안함

이 시각화를 통해 김 양육자는 "구원의 확신은 순조롭게 진행되고 있고,
교제 영역은 좀 더 집중이 필요하다"는 것을 즉시 파악할 수 있습니다.
```

### 2.4 영역별 색상 시스템

**기능 설명**
- 각 영역에 고유한 색상 배정
  - 예: 구원의 확신(초록), 말씀(파랑), 교제(노랑), 증거(빨강) 등
- 그리드 뷰, 대시보드, 진도 표시 등 모든 화면에서 일관된 색상 적용
- 색상의 농도로 진행 상태 표시 (완료: 진한 색, 진행중: 중간 색, 예정: 연한 색)

**사용자 가치**
- **빠른 인지**: 색상만으로 어떤 영역인지 즉시 구분
- **패턴 파악**: 특정 색상이 부족하면 해당 영역이 지연되고 있음을 직관적으로 인식
- **심미적 만족**: 시각적으로 아름답고 정돈된 인터페이스

**사용 시나리오**
```
양육자 대시보드에서 5명의 영혼을 봅니다.

각 영혼마다 색상 바가 표시됩니다:
- 이영희: 초록(많음), 파랑(적음), 노랑(많음), 보라(중간)
  → 한눈에 "말씀 영역(파랑)이 부족하구나" 파악

- 박민수: 초록(많음), 파랑(중간), 노랑(적음), 보라(중간)
  → "교제 영역(노랑)에 집중해야겠다" 판단

색상만으로 5명 전체의 강약점을 3초 안에 파악할 수 있습니다.
```

### 2.5 주차/월차별 상세 내용 관리

**기능 설명**
- 각 주차/월차의 셀을 클릭하면 해당 기간의 상세 양육 내용 표시
- PDF 원본에 기재된 내용 (예화, 성경구절, 활동 내용) 확인 가능
- 완료 체크, 메모 추가, 다음 만남 일정 기록 기능

**사용자 가치**
- **구체적인 가이드**: "이번 주에 무엇을 다루어야 하는지" 명확한 지침 제공
- **준비 지원**: 다음 만남을 위해 필요한 예화, 성경구절 등을 미리 확인
- **기록 보존**: 실제로 다룬 내용, 영혼의 반응 등을 메모로 남겨 추후 참고

**사용 시나리오**
```
김 양육자가 이영희와 이번 주에 만나기 전, GRID를 확인합니다.

"구원의 확신 4주차" 셀을 클릭하면:
- 주제: "신분과 봉헌"
- 성경구절: 요한복음 1:12, 벧전 1:3
- 다룰 내용: "다이아몬드 예화", "왕같은 제사장"
- 이전 만남 메모: "구원의 확신이 흔들릴 때가 있다고 함"

이 정보를 바탕으로 김 양육자는 이영희의 상황에 맞춰 다이아몬드
예화를 준비하고, 구원의 확신을 견고히 할 수 있는 시간을 계획합니다.
```

### 2.6 대시보드 - 전체 양육 현황 요약

**기능 설명**
- 양육자가 관리하는 모든 영혼의 진도를 한 화면에 요약
- 영혼별 전체 완료율, 영역별 진행 상황 표시
- 이번 주 만남 예정자, 지연되고 있는 영역 하이라이트

**사용자 가치**
- **전체 조망**: 여러 영혼을 동시에 관리할 때 누가 어떤 상태인지 한눈에 파악
- **우선순위 설정**: 집중이 필요한 영혼이나 영역을 빠르게 식별
- **시간 관리**: 이번 주 만남 일정을 한곳에서 확인하고 준비 계획 수립

**사용 시나리오**
```
월요일 아침, 김 양육자가 GRID 웹사이트에 접속합니다.

대시보드에 표시되는 내용:
┌─────────────────────────────────────┐
│ 이번 주 만남 예정 (2명)              │
│ - 이영희 (수요일 저녁 7시)           │
│ - 정수진 (금요일 점심 12시)          │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ 전체 진도 현황                       │
│ 이영희 (Convert): 28% 완료           │
│ ████████░░░░░░░░░░░░░░░░░░░░       │
│                                      │
│ 박민수 (Convert): 18% 완료           │
│ █████░░░░░░░░░░░░░░░░░░░░░░░       │
│                                      │
│ 정수진 (Disciple): 42% 완료          │
│ ████████████░░░░░░░░░░░░░░░░       │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ 주의가 필요한 영역                   │
│ • 박민수의 "교제" 영역 (2주 지연)    │
│ • 최지훈의 "말씀" 영역 (3주 지연)    │
└─────────────────────────────────────┘

이를 통해 김 양육자는 이번 주 일정을 확인하고, 특히 박민수와
최지훈에게 집중해야 한다는 것을 인지합니다.
```

### 2.7 진도 기록 및 수정

**기능 설명**
- 각 영역의 주차/월차별로 "완료" 표시 및 취소
- 완료 날짜 자동 기록
- 진도를 잘못 체크한 경우 쉽게 수정 가능
- 건너뛴 주차가 있어도 다음 주차 진행 가능

**사용자 가치**
- **유연한 관리**: 순서대로 진행하지 않아도 실제 진행 상황을 정확히 반영
- **실수 방지**: 잘못 체크한 경우 쉽게 되돌릴 수 있어 부담 없이 사용
- **정확한 기록**: 언제 무엇을 다루었는지 날짜와 함께 보존

**사용 시나리오**
```
김 양육자가 이영희와 만나서 "교제 3주차" 내용을 다루었습니다.

만남 직후, 스마트폰으로 GRID에 접속:
1. 이영희 선택
2. "교제" 영역의 "3주차" 셀 터치
3. "완료" 버튼 클릭
4. 자동으로 오늘 날짜(2026-01-29) 기록
5. 간단한 메모 추가: "찬송교제 좋아함. 다음엔 스포츠 교제 시도"

만약 나중에 실수로 체크했다는 걸 깨달으면:
1. 해당 셀 다시 클릭
2. "완료 취소" 버튼으로 쉽게 되돌리기
```

### 2.8 Convert / Disciple 타입 전환

**기능 설명**
- 한 영혼이 Convert(13주) 과정을 마치고 Disciple(12개월)로 전환될 때 타입 변경
- 타입 변경 시 해당하는 GRID 구조로 자동 전환
- Convert 기간의 기록은 보존하되, 새로운 Disciple GRID 시작

**사용자 가치**
- **연속적인 관리**: 한 영혼의 전체 성장 과정을 끊김 없이 추적
- **기록 보존**: Convert 시절의 양육 기록을 참고하여 Disciple 단계 계획 수립
- **자동화**: 수동으로 새 GRID를 만들 필요 없이 시스템이 자동 처리

**사용 시나리오**
```
이영희가 Convert 13주 과정을 모두 완료했습니다.

김 양육자는 이영희를 Disciple로 전환합니다:
1. 이영희 프로필에서 "Disciple로 전환" 버튼 클릭
2. 확인 메시지: "Convert 기록은 보관되며, Disciple GRID가 생성됩니다"
3. 전환 완료

이제 이영희의 화면에는:
- 상단에 "Convert 13주 완료" 배지 표시
- "Convert 기록 보기" 링크 (과거 기록 확인 가능)
- 새로운 Disciple GRID (12개월 x 14개 영역) 시작

김 양육자는 Convert 시절 이영희가 "말씀" 영역을 좀 더 천천히
진행했다는 기록을 참고하여, Disciple 단계에서도 말씀 영역에
더 신경 쓸 계획을 세웁니다.
```

## 3. 기능 우선순위

### Phase 1 (MVP - 필수 기능)
1. 영혼 관리 (생성, 조회, 삭제)
2. 영역별 독립 진도 관리 (체크/체크 해제)
3. 그리드 뷰 기본 시각화
4. 영역별 색상 시스템

### Phase 2 (주요 기능)
5. 주차/월차별 상세 내용 표시
6. 대시보드 - 전체 현황 요약
7. 진도 기록 및 수정 기능

### Phase 3 (고급 기능)
8. Convert/Disciple 타입 전환
9. 메모 및 만남 일정 관리
10. 통계 및 리포트 기능

## 4. 핵심 차별점 요약

| 기존 방식 (종이 GRID) | GRID 웹서비스 |
|---------------------|--------------|
| 한 번에 하나의 영혼만 볼 수 있음 | 여러 영혼을 동시에 조망 가능 |
| 진도 파악을 위해 일일이 확인 필요 | 색상으로 한눈에 진도 파악 |
| 영역별 진도가 다를 때 관리 복잡 | 영역별 독립 진도 자동 관리 |
| 기록이 지저분해지거나 분실 위험 | 클라우드에 영구 보존 |
| 전체 기간 조망 어려움 | 12개월/13주 전체를 한 화면에 |
| 수정 시 지우개로 지워야 함 | 클릭 한 번으로 수정 가능 |

## 5. 사용자 여정 (User Journey)

### 5.1 새로운 영혼 양육 시작
```
1. 로그인 → 대시보드
2. "새 영혼 추가" 버튼 클릭
3. 이름, 시작일, 타입(Convert/Disciple) 입력
4. 저장 → 자동으로 해당 타입의 GRID 생성
5. 그리드 뷰에서 첫 만남 계획 수립
```

### 5.2 주간 양육 루틴
```
월요일:
- 대시보드에서 이번 주 만남 일정 확인
- 만남 예정인 영혼의 GRID 확인
- 다음 진행할 주차의 내용 미리 보기

수요일 (만남 전):
- 해당 주차 상세 내용 재확인
- 필요한 예화, 성경구절 준비

수요일 (만남 후):
- 완료한 영역/주차 체크
- 만남 내용 메모 추가
- 다음 만남 일정 기록

주말:
- 전체 영혼들의 진도 리뷰
- 다음 주 우선순위 결정
```

### 5.3 장기 점검
```
매월 말:
- 각 영혼의 전체 완료율 확인
- 지연된 영역 파악 및 대책 수립
- 다음 달 목표 설정

분기별:
- Convert → Disciple 전환 대상 확인
- 전체 양육 패턴 분석
- 개선점 도출
```

---

## 6. 기대 효과

### 6.1 양육자 측면
- **효율성 증가**: 여러 영혼 관리 시간 50% 단축
- **체계성 향상**: 누락 없이 모든 영역을 균형 있게 양육
- **기록 보존**: 양육 과정 전체를 영구 보존하여 추후 참고
- **스트레스 감소**: "누구와 무엇을 다루어야 하는지" 고민 시간 감소

### 6.2 영혼(피양육자) 측면
- **맞춤형 양육**: 각자의 속도에 맞는 진도 관리
- **투명성**: 자신의 성장 과정을 시각적으로 확인 가능
- **동기부여**: 완료된 영역이 색상으로 채워지는 것을 보며 성취감
- **연속성**: Convert에서 Disciple로 자연스럽게 이어지는 양육

### 6.3 공동체 측면
- **표준화**: 모든 양육자가 동일한 체계로 양육
- **품질 관리**: 양육 진도와 내용을 가시적으로 관리
- **데이터 축적**: 효과적인 양육 패턴 분석 가능
- **재생산**: 양육받은 사람이 다시 양육자가 될 때 동일한 도구 사용

---

## 7. 성공 지표

- **사용자 수**: 활성 양육자 수
- **관리되는 영혼 수**: 시스템에 등록된 총 영혼 수
- **완료율**: 평균적으로 각 영역이 완료되는 비율
- **지속 사용률**: 3개월 이상 지속적으로 사용하는 양육자 비율
- **전환율**: Convert에서 Disciple로 전환되는 비율
- **만족도**: 양육자의 시스템 만족도 (설문조사)
