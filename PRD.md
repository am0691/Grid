# Part 1: 비전과 철학

## 1.1 제품 비전

### 왜 이 서비스가 필요한가

GRID 웹서비스는 **제자훈련의 본질을 디지털로 옮기는 것**입니다. 단순히 종이를 화면으로 옮기는 것이 아니라, 제자훈련이 가진 '관계 중심의 영적 성장'이라는 핵심 가치를 기술로 구현합니다.

이건교회의 GRID는 오랜 세월 검증된 제자훈련 커리큘럼입니다. CONVERT 과정(13주)에서 DISCIPLE 과정(12개월)으로 이어지는 체계적인 프로그램은 수많은 영혼을 변화시켰습니다. 그러나 이 귀한 훈련이 종이 그리드 한 장에 의존하면서, 시대와 상황의 변화에 유연하게 대응하지 못하는 한계를 보여왔습니다.

**우리의 비전:**

> "모든 양육자가 자신이 돌보는 영혼의 실제 상황에 맞춰, 각 영역을 유연하게 조정하면서도, 훈련의 체계성과 완결성을 잃지 않도록 돕는다."

---

## 1.2 핵심 철학

### 사람은 공장 제품이 아니다 (개별화의 원칙)

모든 영역을 모든 영혼에게 동일한 속도로 주입할 수 없습니다. 한 사람은 기도 영역에서 깊은 은혜를 경험하며 천천히 나아가고, 다른 사람은 증거 영역에서 더 많은 시간이 필요할 수 있습니다.

- 각 영혼의 속도를 존중합니다
- 획일적인 진도보다 내면의 변화를 우선합니다
- 양육자가 영혼의 상태를 관찰하고 조정할 자유를 제공합니다

### 체계성과 유연성은 공존할 수 있다 (균형의 원칙)

GRID의 영역들은 무작위가 아닙니다. 구원의 확신에서 시작해 세계비전으로 확장되는 이 구조는 신학적·교육학적으로 설계된 것입니다. 그러나 순서의 경직성이 오히려 훈련을 방해해서는 안 됩니다.

- 커리큘럼의 완결성을 유지하되, 진행 순서는 조정 가능하게 합니다
- 모든 영역을 다루되, 각 영역의 깊이와 속도는 개인화합니다
- **체계는 레일이 아니라 나침반이어야 합니다**

### 양육은 관계 안에서 일어난다 (관계 중심의 원칙)

제자훈련의 핵심은 콘텐츠가 아니라 관계입니다. 양육자와 영혼 사이의 신뢰, 격려, 책망, 기도가 진정한 변화를 만듭니다. 기술은 이 관계를 대체하는 것이 아니라 **지원**해야 합니다.

- 양육자가 영혼의 진도와 상태를 한눈에 파악할 수 있게 합니다
- 만남의 시간을 행정 업무가 아닌 교제와 나눔에 집중하게 합니다
- **시스템은 뒤에 있고, 사람은 앞에 있어야 합니다**

### 데이터는 섬김의 도구다 (목회적 활용의 원칙)

영혼의 진도 데이터는 평가를 위한 것이 아닙니다. 이것은 양육자가 더 나은 돌봄을 제공하기 위한 통찰의 도구입니다.

- 진도율은 성과지표가 아니라 돌봄의 신호입니다
- 느린 진도는 실패가 아니라 더 많은 관심이 필요한 영역입니다
- 통계는 영혼을 평가하는 것이 아니라 섬기는 방법을 발견하는 것입니다

---

## 1.3 문제 정의: 기존 종이 GRID의 한계

### 경직된 그리드 구조
종이 GRID는 고정된 표입니다. 특정 영역을 건너뛰거나 순서를 바꾸기 어렵습니다. 한 영역에 더 많은 시간이 필요해도 다음 달로 넘어가야 합니다. 결과적으로 "진도를 맞춰야 한다"는 압박감 속에 깊이 있는 훈련보다 피상적인 체크리스트가 됩니다.

### 진도 관리의 어려움
한 양육자가 여러 영혼을 돌볼 때, 각자의 진도를 기억하기 어렵습니다. 종이 그리드는 분실 위험이 있고, 백업이 불가능합니다. 양육자는 만남 전에 "지난번에 뭐 했더라?"를 떠올리느라 귀한 시간을 소비합니다.

### 개인화의 불가능
모든 영혼이 동일한 순서, 동일한 속도로 훈련을 받아야 합니다. 특정 영역에서 더 많은 시간이 필요한 영혼에게 유연성을 제공할 수 없습니다. 획일적인 훈련은 제자훈련의 효과성을 감소시킵니다.

### 협력과 인계의 어려움
양육자가 바뀔 때, 종이 그리드를 물리적으로 전달해야 합니다. 이전 양육자의 메모나 관찰을 충분히 공유하기 어렵습니다. 체계적인 제자훈련 사역이 아니라 개별 양육자의 노력에 의존하게 됩니다.

# Part 2: 핵심 기능

## 2.1 영역별 독립 진도 관리 (핵심 차별점)

GRID 웹서비스의 가장 중요한 기능입니다. 각 영역(구원의 확신, 말씀, 교제 등)마다 **독립적인 진도**를 설정할 수 있습니다.

**기존 방식:**
```
이영희 (Convert 3주차)
→ 모든 영역이 3주차를 진행해야 함
```

**GRID 웹서비스:**
```
이영희 (Convert)
├─ 구원의 확신: 3주차 ← 빠르게 진행됨
├─ 말씀: 1주차 ← 아직 초기 단계
├─ 교제: 3주차 ← 잘 따라오고 있음
├─ 죄에서 떠남: 2주차 ← 좀 더 시간 필요
└─ 참고사항: 2주차
```

**왜 중요한가:**
- 영혼의 실제 이해도와 상황에 맞춘 양육이 가능합니다
- 특정 영역이 지연되어도 다른 영역은 계속 진행할 수 있습니다
- 양육자가 각 영혼에게 최적화된 돌봄을 제공할 수 있습니다

---

## 2.2 색상 기반 그리드 시각화

전체 양육 과정을 한눈에 조망할 수 있는 그리드 뷰를 제공합니다.

```
         │ 구원의확신 │   말씀   │   교제   │ 죄에서떠남 │
         │   (녹색)  │  (파랑)  │  (노랑)  │   (보라)   │
─────────┼───────────┼─────────┼──────────┼────────────┤
 1주차   │     ✓     │    ✓    │    ✓     │     ✓      │
 2주차   │     ✓     │   ★    │    ✓     │    ★      │
 3주차   │    ★     │         │   ★     │            │
 4주차   │           │         │          │            │
  ...

✓ = 완료 (연한 배경색)
★ = 현재 진도 (진한 배경색 + 테두리)
빈칸 = 미래 (회색)
```

**각 영역별 고유 색상:**
- 색상만으로 어떤 영역인지 즉시 구분할 수 있습니다
- 특정 색상이 부족하면 해당 영역이 지연되고 있음을 직관적으로 인식합니다
- 5명의 영혼을 3초 안에 전체 파악할 수 있습니다

---

## 2.3 영혼 관리

양육자가 관리하는 영혼 목록을 생성하고 관리합니다.

- 영혼 등록: 이름, 시작일, 양육 유형(Convert/Disciple) 입력
- 여러 영혼 동시 관리: 한 양육자가 복수의 영혼을 체계적으로 돌봄
- 타입 전환: Convert 완료 후 Disciple로 자연스러운 전환

**사용 예시:**
```
김철수 양육자의 대시보드
├─ Convert
│  ├─ 이영희 (1주차)
│  ├─ 박민수 (3주차)
│  └─ 최지훈 (2주차)
└─ Disciple
   ├─ 정수진 (5개월차)
   └─ 한민준 (8개월차)
```

---

## 2.4 완료 체크 및 메모

각 영역의 주차/월차별로 완료 표시와 메모를 기록합니다.

- 셀 클릭 → 완료 체크 + 메모 입력
- 완료 날짜 자동 기록
- 실수로 체크한 경우 쉽게 수정 가능
- 건너뛴 주차가 있어도 다음 주차 진행 가능

**왜 중요한가:**
- "이영희가 지난번에 다리 예화를 잘 이해했다"는 기록을 남기면
- 다음 만남에서 그 위에 더 깊은 이야기를 나눌 수 있습니다
- 양육의 연속성과 깊이가 보장됩니다

---

## 2.5 대시보드

양육자가 관리하는 모든 영혼의 현황을 한 화면에 요약합니다.

```
┌─────────────────────────────────────┐
│ 이번 주 만남 예정                    │
│ • 이영희 (수요일 저녁)               │

┌─────────────────────────────────────┐
│ 주의가 필요한 영역                   │
│ • 박민수의 "교제" 영역 (2주 지연)    │
│ • 최지훈의 "말씀" 영역 (3주 지연)    │
└─────────────────────────────────────┘
```

# Part 3: 사용자 경험

## 3.1 사용자 여정

### 새로운 영혼 양육 시작
1. 로그인 → 대시보드
2. "새 영혼 추가" 버튼
3. 이름, 시작일, 타입 입력
4. 저장 → 자동으로 GRID 생성
5. 그리드 뷰에서 첫 만남 계획 수립

### 주간 양육 루틴
```
월요일:
  대시보드에서 이번 주 만남 일정 확인
  → 만남 예정 영혼의 GRID 확인
  → 다음 진행할 주차의 내용 미리 보기

만남 후:
  완료한 영역/주차 체크
  → 만남 내용 메모 추가
  → 다음 만남 계획
```

---

## 3.2 핵심 화면

### 대시보드
- 모든 영혼을 카드 형태로 표시
- 각 카드에 진도율, 현재 주차, 영역별 색상 요약
- 빠른 필터: Convert / Disciple

### 그리드 뷰 (핵심 화면)
- 세로축: 주차(1~13) 또는 월(1~12)
- 가로축: 영역 (Convert 5개, Disciple 14개)
- 각 셀: 완료/현재/미래 상태를 색상으로 표시
- 셀 클릭: 완료 체크 + 메모 입력

### 영혼 상세 정보
- 기본 정보 편집
- 메모 히스토리
- 전체 진도 요약 (영역별 완료율)

---

## 3.3 사용성 원칙

### 직관성
- 주요 기능까지 3클릭 이내
- 색상 + 아이콘 + 텍스트 조합으로 상태 명확히 표시
- 범례를 항상 표시하여 색상 의미 확인 가능

### 접근성
- 다양한 연령대(20대~60대) 고려
- 높은 명암 대비, 큰 클릭 영역
- PC 우선, 기본 모바일 지원

### 신뢰성
- 자동 저장으로 데이터 손실 방지
- 클라우드 백업
- 실수 시 쉬운 복구

# Part 4: GRID 구조

## GRID FOR CONVERT (13주)

**키워드:** Joy (사 40:1-3)

**5개 영역:**
| 영역 | 핵심 목표 |
|------|----------|
| 구원의 확신 | 예수님의 구원을 확신하고 기뻐하는 삶 |

| 교제 | 하나님의 가족의 일원이 됨과 교제의 특권을 즐거워함 |
| 죄에서 떠남 | 성령의 능력으로 구습을 벗어버리고 새 사람을 입기를 즐거워함 |
| 참고사항 | 양육 시 주의사항 및 참고 내용 |

# Part 5: 성공 지표

## 양육자가 만족한다
- "이제 종이로 돌아갈 수 없다"는 피드백
- 행정 시간 50% 감소, 교제 시간 증가

## 영혼이 완주한다
- CONVERT → DISCIPLE 이탈률 30% 감소
- 훈련 만족도 향상

## 교회가 성장한다
- 체계적인 제자훈련 데이터 확보
- 목회적 의사결정의 근거 마련

## 기술이 사라진다
- 사용자가 "시스템"을 의식하지 않음
- 자연스럽게 훈련에 집중
- **기술은 뒤에서 조용히 지원**

---

# Part 6: 결론

## 우리의 꿈

양육자가 영혼과 만날 때, 종이와 펜을 꺼내 "지난번에 뭐 했더라?"를 고민하는 대신, 따뜻한 눈빛으로 "이번 주는 어떠셨어요?"라고 물을 수 있는 세상.

영혼이 12개월의 여정을 마칠 때, 체크된 표가 아니라 변화된 삶을 보며 감사하는 세상.

교회가 제자훈련 사역을 계획할 때, 추측이 아니라 통찰을 바탕으로 결정하는 세상.

**기술이 사람을 대체하는 것이 아니라, 사람이 사람에게 더 집중할 수 있도록 섬기는 세상.**

---

> **"그러므로 너희는 가서 모든 민족을 제자로 삼아"** (마태복음 28:19)
>
> 이 명령을 21세기 방식으로 순종하는 것, 그것이 GRID 웹서비스의 존재 이유입니다.

## 용어 정리

| 용어 | 의미 |
|-----|------|
| 양육자 (Leader) | 영혼을 양육하는 사람 |
| 영혼 | 양육을 받는 사람 (Convert 또는 Disciple) |
| Convert | 13주 초기 양육 과정에 있는 사람 |
